# Formatting (Communication rules)

## Formatting is communication
- Readability outlives functionality.
- Team rules matter more than personal preference: agree and automate.

## File structure like a newspaper
- Top: most important concepts with least detail
- Bottom: low-level helpers and implementation details

## Vertical distance
- Keep related concepts close.
- Put callers above callees.
- Avoid hiding important fields halfway down the file.

## Practical default
- Small files are easier to understand.
- Use whitespace to separate concepts, and keep related lines dense.
